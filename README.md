# git-showcase

A dedicated sandbox repository for exploring, demonstrating, and mastering Git features and GitHub workflows.

## Purpose

This repository serves as a practical playground for testing various Git-related concepts in a controlled environment. It is designed to showcase:

- **GitHub Workflows:** CI/CD pipelines, automated testing, and custom actions (e.g., cascade merges).
- **Branching Strategies:** Exploring Feature branching, GitFlow, and trunk-based development.
- **Versioning & Tagging:** Managing releases and semantic versioning.
- **Conflict Resolution:** Simulating and resolving complex merge and rebase conflicts.
- **Git Internals:** Experimenting with cherry-picking, interactive rebasing, and stashing.

## Getting Started

Feel free to fork this repository, create branches, and experiment with PRs to see the workflows in action.

## Wishlist

Future explorations and additions planned for this sandbox:

- **Git Hooks:** Implementing custom pre-commit and pre-push hooks (e.g., for linting or secret scanning).
- **Conventional Commits:** Automating versioning and changelog generation using `semantic-release` or similar tools.
- **Git LFS:** Showcasing how to handle large files efficiently.
- **Advanced GitHub Actions:** Environment-specific deployments, matrix builds, and reusable workflows.
- **Repository Templates:** Setting up `.github` configurations (PR templates, issue templates, and contributing guidelines).
- **Git Submodules & Subtrees:** Managing dependencies and external code within the project.
- **GPG Signing:** Demonstrating signed commits for verified repository integrity.
